# Multiagent Story Generation System

A complete multiagent system for generating executable story graphs for GTA San Andreas using hierarchical GEST refinement with GPT-5 and LangGraph.

## Overview

This system generates story graphs through 5 levels of progressive refinement:

- **Level 0**: Abstract Narrative GEST (pure narrative with semantic correlations)
- **Level 1**: Scene-Level GEST (scenes with location types and objects)
- **Level 2**: Region-Specific GEST (mapped to game regions)
- **Level 3**: Actor-Choreographed GEST (detailed per-actor actions)
- **Level 4**: Game-Executable GEST (final JSON format for MTA)

Each level is generated by a specialized agent using GPT-5, and the final Level 4 GEST is validated by actually running it in the MTA game engine. If validation fails, an error correction loop attempts to fix issues.

## Installation

1. **Install Python dependencies:**

```bash
cd multiagent_story_system
pip install -r requirements.txt
```

2. **Configure OpenAI API key:**

Create a `.env` file in the project root:

```bash
# Copy the example file
cp .env.example .env

# Edit .env and add your OpenAI API key
# OPENAI_API_KEY=your-actual-api-key-here
```

Get your API key from: https://platform.openai.com/api-keys

3. **Copy documentation and examples:**

```bash
python setup/copy_documentation.py
```

## Quick Start

### 1. Export Game Capabilities

First, export the game world capabilities from MTA:

```bash
python main.py --export-capabilities
```

This will:
- Start the MTA server in EXPORT_MODE
- Extract all actions, objects, and regions from the game
- Generate `game_capabilities.json` in the sv2l resource
- Copy the file to `data/game_capabilities.json`
- Automatically shut down the server

### 2. Generate a Story

Generate a story from scratch (creative mode):

```bash
python main.py
```

Or with narrative seed sentences (guided mode):

```bash
python main.py --seeds "Two friends meet at a bar" "A conflict arises over a misunderstanding"
```

### 3. View Results

Generated stories are saved to `output/generated_graphs/`:

- `story_TIMESTAMP_L0.json` - Abstract narrative
- `story_TIMESTAMP_L1.json` - Scene breakdown
- `story_TIMESTAMP_L2.json` - Region mapping
- `story_TIMESTAMP_L3.json` - Actor choreography
- `story_TIMESTAMP_L4.json` - Game-executable (ready for MTA)

Videos are saved to `output/videos/` (if validation succeeds).

## Configuration

Edit [config.yaml](config.yaml) to customize:

- OpenAI model and parameters (temperature, max_tokens)
- MTA server paths and settings
- Validation thresholds and retry limits
- Logging levels and output formats
- Development mode settings

## Architecture

### Agents

- `NarrativeAgent` - Generates Level 0 abstract narrative
- `SceneBreakdownAgent` - Refines Level 0 → Level 1
- `RegionMappingAgent` - Refines Level 1 → Level 2
- `ActorChoreographyAgent` - Refines Level 2 → Level 3
- `GameGESTGeneratorAgent` - Refines Level 3 → Level 4
- `ValidationAgent` - Validates Level 4 via MTA simulation
- `ErrorCorrectorAgent` - Corrects Level 4 based on errors

### Workflow

The LangGraph workflow orchestrates agents in sequence:

```
Level 0 → Level 1 → Level 2 → Level 3 → Level 4 → Validation
                                                         ↓ (if failed)
                                              Error Correction
                                                         ↓
                                                   Validation
                                                    (retry up to 3x)
```

### Utilities

- `FileManager` - Handles all file I/O for GEST files and configurations
- `MTAController` - Manages MTA server process (start/stop/configure)
- `LogParser` - Parses MTA logs for validation results

## Advanced Usage

### Resume from Checkpoint

LangGraph checkpoints workflow state. Resume a previous run:

```bash
python main.py --thread-id my-story-1
```

### Development Mode

Skip expensive MTA simulation during development:

```bash
python main.py --skip-simulation
```

Or enable in [config.yaml](config.yaml):

```yaml
development:
  skip_simulation: true
```

### Verbose Logging

Enable DEBUG-level logging:

```bash
python main.py --verbose
```

### Save Intermediate Outputs

Enable in [config.yaml](config.yaml) to save GEST at each level:

```yaml
development:
  save_intermediate_outputs: true
  intermediate_output_dir: "output/intermediate"
```

## Validation

The system validates generated stories by:

1. Saving Level 4 GEST to file
2. Configuring MTA server in SIMULATION_MODE
3. Running the actual game simulation
4. Parsing server/client logs for errors
5. Checking video output was generated

Validation succeeds if:
- Simulation completes without errors
- All actions execute successfully
- Video file is generated

If validation fails:
- Error messages are extracted from logs
- ErrorCorrectorAgent analyzes and fixes issues
- Validation re-runs (up to 3 attempts total)

## Troubleshooting

### Game capabilities not found

Run `python main.py --export-capabilities` first.

If the export completes but the file still isn't found:
1. Check if the file exists in `../sv2l/game_capabilities.json`
2. Manually copy it if needed: `cp ../sv2l/game_capabilities.json data/`
3. Verify the `paths.game_capabilities_source` setting in [config.yaml](config.yaml)

### OpenAI API errors

Ensure your `.env` file exists with a valid `OPENAI_API_KEY`. Check that:
- The `.env` file is in the project root directory
- The API key is valid and has sufficient credits
- There are no extra spaces or quotes around the key

### MTA server not starting

Check paths in [config.yaml](config.yaml) match your MTA installation.

### Validation always failing

1. Check `logs/validation.log` for details
2. Try `--skip-simulation` to test generation without validation
3. Review MTA server logs at `{server_root}/mods/deathmatch/logs/`

### Out of token errors

Increase `max_tokens` in [config.yaml](config.yaml) for specific agents.
